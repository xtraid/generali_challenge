How the Models Actually Work

Collaborative Filtering

Algorithm: Cosine similarity on co-occurrence matrix

Example: Coverage 001 → 057 has 86% similarity

Handles: 10M+ records via sparse matrices

Profile-Based ML

Algorithm: 15 Random Forests (100+ trees each)

Logic: IF value>€15k AND claims=0 → 80% prob

Features: 21 engineered predictors

Hybrid Combiner

Formula: 0.6×collab + 0.4×profile

Fallback: Auto-switches to profile if cold start

Validated: Cross-tested for optimal weights

Technical Innovations

  • Sparse Matrix Optimization: Handles billions of combinations without memory overflow
  • Binary Classification Strategy: 15 separate YES/NO models instead of one multi-class model = better multi-recommendation
  • Smart Weighting: 60/40 ratio found through extensive cross-validation testing
  • Real-time Performance: <20ms response time per customer recommendation
